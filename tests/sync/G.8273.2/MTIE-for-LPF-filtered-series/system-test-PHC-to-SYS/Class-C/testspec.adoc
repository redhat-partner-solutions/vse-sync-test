= G.8273.2 MTIE-for-LPF-filtered-series system-test-PHC-to-SYS Class-C (System-level)

https://www.itu.int/rec/T-REC-G.8273.2/en[G.8273.2 Section 7.2 Table 2] specifies
requirements on the accuracy of the MTIE for
https://www.itu.int/rec/T-REC-G.8273.2/en[Class-C] under normal, locked conditions.
This is a system-level test that validates the complete end-to-end path from
PHC to System Clock (SYS). The purpose of this test is to verify that the
aggregate MTIE at the system output meets the full Class-C requirement.

IMPORTANT: It is critical to note that the results of this test are only valid
when both the test environment and the System Under Test (SUT) have been
independently validated. While requirements on their acceptability are not
defined here, it is noted that they include timestamping accuracy.

== Goal

Verify that the MTIE at the system clock output (PHC-to-SYS) under
normal, locked operating conditions meets 100% of the overall MTIE requirement
for https://www.itu.int/rec/T-REC-G.8273.2/en[Class-C].

== Scope

* Ignore samples in an initial transient window
* Verify samples in the test window
* System-level validation of end-to-end MTIE

== Out of scope

* Independent validation of the test environment
* Independent validation of the SUT
* Provision SUT

== Acceptance criteria

1. All filtered MTIE samples during each observation interval
   are less than 100% of overall MTIE accuracy requirement
2. The time output is locked for the test window
3. The test window is at least 1000 seconds long

== Test procedure

1. Establish normal, locked conditions per
   https://www.itu.int/rec/T-REC-G.8273.2/en[G.8273.2 Section 7]
2. Capture at least 1000 seconds of `phc2sys` log data
3. Verify log data meets acceptance criteria
